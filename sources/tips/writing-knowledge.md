# Tips for writing knowledge files

> Sample knowledge for the search demo. Replace with your team's first-hand know-how.

- One topic per file — the search ranks whole files, so focused files rank cleanly
- Lead with the conclusion
- Write pitfalls as "symptom -> cause -> fix", because people search by the symptom's wording
- Prefer first-hand findings over things already covered by external docs — that is the knowledge no one else has
